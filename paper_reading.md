Attention is all you need 的核心创新点：Transformer架构，Self-Attention自注意力机制
这两个是什么，现在先不管。现在的思路即解释这两个词是什么，以及为什么要这么做（为什么是时代分水岭）

其实就是回答这两个问题
1.以往模型的结构和处理逻辑，存在的问题，以及为什么有这些问题（结构导致问题）
2。这两个创新点解决了什么，以及怎么解决的

一切一切的开始，我们先要理解一个东西
怎么把各种东西的信息数学化，变成计算机可以理解的形式，就是数字向量
每个词通过很多维度的特质被表现出来，比如x轴是国家，y轴是城市重要性，那巴黎——法国，和，北京——中国这两个向量的夹角应该非常小，因此点积越大，越小则代表关系差别越大

我们先来讲讲最简单的前馈神经网络（FNN feedback neural network）
输入层x —— w权重矩阵v —— 输出层y （只能接受输入一个向量，输出一个向量）
但是不适合有序列的任务（不适合说话）
翻译我吃饭：I have food
我吃饭（三个token）
词向量表示（Embedding）
我:[ 0, 2, 3]
吃:同理
饭:同理
但是有问题：如果把向量的各个值平均输入，就完全失去了顺序意义
           如果强行拼接成一个9维向量，输入层越来越长，效率低

所以有了RNN
输入层x —— w权重矩阵v（加了一个u：h循环在x 和 y之间） —— 输出层y
第一个时刻，只输入一个token
第二个，第三个同理
时刻1的输出结果会变成时刻2的输入层的一部分，之后同理，递归的姿态
t1：（我x1 * w + h0 * u）*g = h1  同时有了y1 （g为激活函数）
t2：同理 吃x2 h1 h2 
不过要是我翻译过来不是单词和汉字不是一一对应怎么办，比如我吃晚饭：I have dinners
所以有了编码器和解码器
编码器：不输出y了，改为得到一个c（上下文向量，c=h3）（c其实已经包含了所有信息）
解码器：c作为输入层，“逆运算一下”得到s1，s2，等等，其中每一个s都由上一个s和c共同计算得到（其实这里没太搞懂，为什么就解决了不一一对应的问题）
不过经过这么多次运算，x1等早期信息对c的影响肯定会比x3等最后几个影响弱，俗称搞忘了（遗忘问题）
还有一个就是：实际上每个时刻的重要性是不一样 

所以有了注意力机制
其实就是把c区分出c1，c2，等等
比如：c0 = 0.1h1我 + 0.1h2吃 + 0.4h3晚 + 0.4h4饭
     让他注意到晚饭连在一起的，才能翻译出dinner，不然就有可能是late rice
就解决了以上两个问题

以前在处理问题时，比如翻译问题，常用序列转导模型（sequence transduction model）
    就是说：要处理带顺序的数据，且数据很重要（比如说话：我吃饭，饭吃我，肯定是两个意思）
    这个序列转导模型主要基于RNN和CNN（其实一般用的是RNN）
    （RNN（循环神经网络）：专门处理：序列数据、有先后顺序的东西 网络会把前面的信息 “循环” 传给后面，有记忆。
      处理一句话：
     “我今天没带伞，所以被____了”
      机器要记住前面“没带伞”，才知道后面填 “淋湿”
      它是一环扣一环，像链条一样循环传递信息）
      （CNN（卷积神经网络）：专门处理：图片、图像、空间信息 用一个小窗口，在图片上滑来滑去，提取特征。（卷：翻转、滑动 积：相乘、累加 卷积 = 一边滑动，一边乘加）
        就像你拿放大镜在图片上一点点扫
         扫的时候，看：
         有没有边缘
         有没有纹理
         有没有角、线条
         扫完整张图，就知道这是猫还是狗、是人还是车）
    通常包括一个encoder和一个decoder
    其中有一些会用到注意力机制

RNN还有个问题就是只能等待上一步处理完，才能做下一步，不能并行式计算
不过现在有了Transformer,一个仅仅依靠注意力机制的模型，不用RNN,CNN,（还是有encoder和decoder）
我们注意到机器学习时百分百会给它做有答案的题来训练，不需要等他一个一个的做，他可以直接背答案，不走流程
我们现在叫他不要等了
每个词转化从一个512维的向量
假设在编码“晚”时，让他以一个全局的视角，看到剩下所有的词，把这些信息编码到晚这个词本身的语义向量里，同时对其他的token也做同样的工作
不过不像RNN，没有时序信息，所以需要一个额外的编码表示每个词的位置信息
于是得到一个十分完整的向量
他翻译的时候，他就可以看见一部分信息，直接生成下一个，反正在背答案嘛
不过这个只有在训练时可以这样

在一次典型的训练中，我吃晚饭变成四个向量，可以被计算机理解
通过余弦判断相似度
位置编码，每个词都有一个tag表示相对位置（数学原理看不懂）
上下文编码：有三个权重矩阵wq,wk,wv，每个词单独拿出来，分别乘以三个矩阵，最终得到三个1 * 512 的向量(query查询,key键,value值) q矩阵，k矩阵，v矩阵（每个词都有）
           如果饭根据上下文，向一日三餐靠拢，而不是向米饭靠拢，就翻译的更好
           q发问，k检索，v真正的这个词的意思
           饭q和饭k做点积，比如q是“我是名词吗”，只要点积越大，就说明越是，不是就越小，
           经过注意力公式变成多个概率之和为1的情况，相当于权重，
           最后在去和k做点积，得到全新的向量，变成一个包含上下文信息的向量
这一大堆就是一个单头注意力的过程：输入的4 *512    经过三个512 * 512 的矩阵 ，得到输出的4 * 512 的矩阵
多头注意力，三个矩阵改成512 * 64 得到4 * 64 （8组）
           将8组拼接还得到4 * 512 
           每一个方向都代表一个特征（比如：甜，不甜  身高：200——0）
           8个头每个关注不同的领域，比如一个关注口味，一个关注身高，每个头都在一个小空间计算
那为什么要多个头呢？因为每个头可以不同的query，比如我是名词吗，后面有谓词吗（每个头关注不同的视角，比如一个关注语法关系，一个关注代词逻辑，一个关注情绪色彩）（不过模型并不会告诉你，它关注哪一个，它完全以数学方法自动发现的8种视角，人的语言可能说不清楚）
另外，拼接后要额外乘上一个线性层，相当于把8个低清图片拼成一个高清图片，如果没有这步，还是低清
中间还有一个残差链接（输出 = 层计算结果 + 原始输入）和归一化（对每一个样本，把它的数值：拉到 均值接近0 方差接近1）就是让模型分布更好看，更稳定
最后还是一个FNN
输出时有个masked multi-headed attention，掩码机制，就是说不能让他抄答案，需要把其中一些信息的权重设置的非常小或者直接等于0
自注意力就是说，qkv全部都来源与自己，关注于自己

再着重复习一下qkv什么意思：
我们把数据里的每个单词都想象成一小人，这些小人需要了解一下对方，每个人都有两个问题（我的身份是什么K，我想知道什么Q）
例：我吃饭，它里面有肉
K:“我”是什么：人称代词，主语 Q:我做了什么
句子里面每个小人都应该计算出他的Q和K
（每个单词最关心的问题，QK早在训练时就确定）
接下来，每个词的Q都要和每个词的K做点积运算，看看有没有合适的
比如：“它”的Q是：我是什么。  它一看“我”的K是人称代词，那肯定就不是这个“我”了，所以它就不在这上面花注意力（权重降低）。但是它一看“饭”，“饭”的K说，我是一个名词，一个物品，它就加大注意力，于是它就会查看饭的V
(V是由很多种向量在不同权重的分配，比如：饭80%概率是指一日三餐，20%概率是指大米饭)查看后，“它”就会向一日三餐偏移，于是模型便理解了它
（不过这都是数学向量计算）

为什么这样就是更好
1.每层的总复杂度降低了
2.具有并行计算能力
3.模型内部学校长距离依赖的问题（注意力机制：防遗忘）

为什么 Self-Attention 可以实现训练并行化？它和 RNN 的本质区别是什么？
RNN 的核心逻辑是序列串行计算：每个 token 的隐藏状态输出，必须依赖前一个 token 的隐藏状态计算结果，因此无论训练还是推理，都只能按序列顺序逐个计算，无法利用 GPU 的并行算力，长序列训练效率极低。
而自注意力机制的计算逻辑完全不同：序列中每个 token 的输出，都是通过与序列内所有 token 的 Q、K、V 做矩阵运算得到的，每个 token 的计算过程相互独立，不需要等待其他 token 的计算结果。因此训练时，整个序列的注意力计算可以一次性并行完成，充分利用 GPU 的并行算力，训练速度相比 RNN 有数量级的提升。这里我也厘清了一个关键误区：Transformer 在推理阶段依然是自回归的，需要逐个生成 token，无法并行；但训练阶段的并行优势，是它能替代 RNN、支撑大模型规模化训练的核心原因。
